## ShadowCacheInvalidationBehavior Objects

```python
class ShadowCacheInvalidationBehavior(EnumBase)
```

Setting to control shadow invalidation behavior (in particular with respect to Virtual Shadow Maps and future methods).

**C++ Source:**

- **Module**: Engine
- **File**: SceneTypes.h

<a id="unreal.ShadowCacheInvalidationBehavior.AUTO"></a>

#### AUTO

0: Default. Invalidates based on World Position Offset material, and transform changes, attempting to do the right thing given the information available to the renderer.

<a id="unreal.ShadowCacheInvalidationBehavior.ALWAYS"></a>

#### ALWAYS

1: Always invalidate shadows, can be used to flag a primitive that is using some method of animating that is not known to the system.

<a id="unreal.ShadowCacheInvalidationBehavior.RIGID"></a>

#### RIGID

2: Suppresses invalidations that would otherwise be generated by e.g., World Position Offset (WPO). Can be used for example when they use WPO statically used, or the artifacts are subtle enough to not warrant the performace overhead (use with care).

<a id="unreal.ShadowCacheInvalidationBehavior.STATIC"></a>

#### STATIC

3: In addition to "Rigid" behavior, also suppress invalidations due to transform changes. Add/Remove will still trigger invalidations.
If the primitive is actually moved or animated somehow the visual result is undefined.
